1. 플랫폼 : 개발을 할 수 있는 기반 프로그램이 설정되어있는 환경
ex > 리눅스 os 위에 db나 노드 js 같은 IDE와 SDK가 전부 깔려있는 개발 환경. 플랫폼위에서 개발을 한다.
클라우드 iaas하면서 했던것이 모두 플랫폼 만드는 것

2. 프레임 워크 : 프로그래밍 언어에서 프로그래밍을 할 때 특정 틀을 가져서 누구나 쉽게 알아보고 
똑같이 코딩할수 있게 하는 틀. 특별한 기능을 가지고 있지는 않다. 
자바 스프링은 프레임 워크이다. 자바라는 틀에서 벗어나지 않았고 그저 코딩하는 방식이나 제작하는 방식
이 정해져있다. 하지만 노드 js는 프레임 워크가 아니다. 자바스크립트지만 그것과는 다른 해석방식이
필요하기때문에 단순 프레임 워크가 아니다.

3. 라이브러리 : 라이브러리는 개발을 할 수 있는 클래스 , 함수들을 기본적으로 제공해준다.
개발자는 이를 다운 받아서 이용할 수도 있고 코드 수정을 할 수도 있다. 재사용성 좋다.

4. API 개발 인터페이스 : 프로그래밍을 할 때 라이브러리 안의 클래스는 매소드라는 특정 인터페이스를 
제공해 준다. 인터페이스는 라이브러리를 이용할 수 있게 해주는 매개체이다. 라이브러리를 사용하려면
API를 이용한다.
